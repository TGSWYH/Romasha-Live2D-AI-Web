# 🌸 Romasha 幻影：Web 端 Live2D 陪伴与视觉小说推演引擎

**基于大型语言模型（LLM）、WebSocket、TTS 与 Live2D 的沉浸式网页虚拟生命项目。**

---

## 【写在前面的话】

这个项目源于我对 RPG MAKER 游戏《Princess Synergy》以及女主角 **Romasha** 的喜爱。

我把她的 Live2D 模型提取出来，接入了 AI、大语言模型、语音生成、酒馆式世界书机制、长期记忆、自动推进剧情与地图切换系统，最终做成了这个 **可在网页/移动端运行的 Romasha 陪伴系统**。

这不是一个单纯的网页聊天框，也不是一个“套个模型皮肤”的对话机器人。
她拥有：

- 相对独立的人格与情绪逻辑
- 基于《Princess Synergy》的世界观与章节推进系统
- 长期记忆与动态 Lorebook
- 地图 / 场景认知能力
- Live2D 物理反馈
- 语音、口型、剧情推演联动

在这个项目里，Romasha 不只是和你聊天。
她会记住你、感知你、回应你，甚至在你干预下，**重新书写属于她的命运**。

---

## ✨ 【核心功能概览】

### 1. 📖 视觉小说自动推演模式
Web 端完整支持 Romasha 的剧情推演系统。

- **章节推进机制**：随着剧情前进，不同章节的地图、角色与世界信息逐步解锁
- **命运抉择系统**：在关键节点自动生成 3 个选项，并带倒计时
- **自定义改命**：不满意系统给的选项？你可以直接输入自己的抉择
- **4 档参与度**：
  - `0` 纯旁观
  - `1` 轻度参与
  - `2` 中度同行
  - `3` 深度羁绊主角

你不只是看故事，而是在**介入她原本悲剧性的世界线**。

---

### 2. 🧠 动态世界书与长期记忆
- **ChromaDB 向量记忆**：记录你与她之间的重要对话与经历
- **剧情摘要压缩机制**：旧对话会自动压缩整理，避免上下文无限膨胀
- **动态世界书（Lorebook）**：人物关系、称呼、道具、地点等会被 AI 自动更新
- **章节防剧透机制**：世界书和地图都带解锁控制，避免模型提前知道未来剧情

简单说，她不只是“记住一句话”，而是在**逐渐形成持续性的世界理解**。

---

### 3. 🗺️ 地图与空间感知系统
- 知道当前所在地点
- 可根据剧情自主输出 `[move_to_地点]` 进行场景转移
- 地图地点随章节推进逐步开放
- 支持地点 Lore 注入，让模型知道“自己现在在哪里、这里该发生什么”

这使得她的行为不再悬浮，而是更像真实地生活在一个具体空间里。

---

### 4. 👀 Live2D 物理互动
Web 端同样保留了大量物理交互能力：

- **视线跟随**
  - 触控跟随
  - 陀螺仪跟随
  - 休眠关闭
- **触碰反馈**
  - 头部
  - 脸颊
  - 胸部
  - 腰部
  - 臀部
  - 手部
  - 腿部
- **动作 / 情绪联动**
  - 摸不同位置会触发不同情绪动作与台词
  - 支持瞬时动作与常驻情绪的优先级切换

---

### 5. ⏰ 现实时间生物钟与自动换装
她会根据现实时间改变状态：

- 白天：日常制服
- 傍晚：连衣制服
- 夜晚：睡衣
- 深夜高亲密度：可能切换为浴巾状态
- 节假日：自动穿上特殊服饰

并且玩家主动换装后，系统会进行一定的生物钟锁定，避免“刚换完又被时间机制强制换回去”。

---

### 6. 🗣️ 语音、口型与双 TTS 引擎
支持：

- **CosyVoice**
- **GPT-SoVITS**

能力包括：

- 文本转语音
- 实时口型同步
- 语音中插入呼吸、叹息等声音标签
- 可选静默翻译成日语后再发音
- 自动关闭失败语音服务，避免前端卡死

---

### 7. 💤 发呆、自言自语与挂机反应
如果你长时间不操作，她不会像普通机器人一样彻底静止：

- 会移开视线
- 进入发呆状态
- 小概率碎碎念
- 在高亲密度和特殊服装状态下可能触发隐藏彩蛋

她不是在“等输入”，而是在“继续活着”。

### 可以做什么
你可以把本项目上传到 GitHub 用于：

- 开源源码
- 展示 README
- 管理版本更新
- 提交 issue / 反馈
- 让别人 clone 后本地部署

### 不能直接做什么
**本项目不能直接完整运行在 GitHub Pages。**

原因：

1. GitHub Pages 只能托管静态页面
2. 本项目依赖 FastAPI 后端
3. 需要 WebSocket 长连接
4. 需要本地/远程 LLM 与 TTS 接口
5. 需要写入 `config.json`、`world_data/`、记忆数据库等文件

### 正确理解
- **GitHub 仓库**：放源码
- **实际运行环境**：本地电脑 / 云服务器 / VPS / 局域网主机

## 🛠️ 【安装与部署指南】

## 1. 基础环境
请确保你已经安装：

- Python 3.8 或更高版本
- pip
- 可用的 LLM 接口（OpenAI 兼容或 Ollama）
- 可选的 TTS 服务（CosyVoice / GPT-SoVITS）

---

## 2. 安装依赖
在项目根目录执行：

```bash
pip install -r requirements.txt
```

---

## 3. 准备前端模型资源
由于版权与模型作者权益原因，本仓库**不直接提供 Live2D 模型文件**。

请自行准备并放入以下目录：

```bash
web/
└── model/
    └── romasha_talk.model3.json
```

同时确保相关贴图、motions、expressions、moc3 等资源路径正确。

如果你使用的是本仓库附带的 `web/index.html`，默认会加载：

```text
/model/romasha_talk.model3.json
```

---

## 4. 初始化配置文件
首次运行时，系统会自动生成 `config.json`。

你需要填写或修改这些字段：

```json
{
  "api_type": "openai",
  "api_key": "你的密钥",
  "base_url": "你的接口地址",
  "target_model": "你的模型名",

  "voice_enabled": true,
  "tts_engine": "cosyvoice",

  "cosy_url": "http://127.0.0.1:9880/api/tts",
  "cosy_character": "Romasha",

  "sovits_url": "http://127.0.0.1:9880/",
  "sovits_ref_audio": "参考音频路径",
  "sovits_ref_text": "参考文本",
  "sovits_ref_lang": "ja",
  "sovits_target_lang": "ja"
}
```

---

## 5. 启动服务
运行：

```bash
python server.py
```

启动后默认监听：

```text
http://127.0.0.1:8000
```

局域网访问时可用：

```text
http://你的本机IP:8000
```

## 🚀 【使用说明】

### 打开网页
浏览器访问：

```text
http://127.0.0.1:8000
```

如果是手机访问，请保证手机和运行后端的设备在同一局域网下。

---

### 基础交互
- 点击右下角 `💬` 打开输入框
- 直接输入文字与她聊天
- 点击模型身体不同位置可触发不同触摸反馈
- 双指可缩放模型
- 拖动可移动模型位置

---

### 剧情推演面板
点击右上角：

```text
📖 展开剧情推演
```

可使用功能：

- 设置你的名字
- 选择参与度
- 开始 / 停止 / 继续推演
- 选择命运分支
- 输入自定义抉择
- 下载当前推演日志
- 退出剧情模式

---

## ⌨️ 【内置指令】

### 剧情类
```text
/auto 2
```
开启剧情推演模式，数字代表参与度：

- `0` 纯旁观
- `1` 轻度参与
- `2` 中度同行
- `3` 深度羁绊

```text
/chapter 2
```
强制跳到指定章节。

```text
/choice 你的选择
```
在推演模式中手动输入你的抉择。

---

### 系统类
```text
/reset
```
重置长期记忆、剧情摘要、动态世界书与亲密度。

```text
/touch 0
/touch 1
```
关闭 / 开启物理触碰。

```text
/track 0
/track 1
```
关闭 / 开启视线跟随。

```text
/voice 0
/voice 1
```
关闭 / 开启语音。

```text
/voice cosyvoice
/voice sovits
```
切换 TTS 引擎。

```text
/size 0
/size 1
/size 2
```
切换气泡字号。

```text
/ja 0
/ja 1
```
关闭 / 开启静默日语翻译后发音。

├── world_info.py             # 世界观基础设定

├── test_motions.py           # 动作/换装调试台
